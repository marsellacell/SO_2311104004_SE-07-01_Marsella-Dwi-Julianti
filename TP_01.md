## Dasar Teori

Sistem operasi adalah perangkat lunak yang berfungsi sebagai penghubung antara perangkat keras (hardware) dan pengguna (user). Sistem operasi mengatur pengelolaan proses, memori, sistem file, serta komunikasi antar perangkat dalam sebuah komputer.

Pada praktikum ini digunakan sistem operasi Xinu (Xinu Is Not Unix), yaitu sistem operasi kecil yang digunakan untuk tujuan pembelajaran konsep dasar sistem operasi seperti manajemen proses, semaphore, memori, dan sistem pemanggilan (system call).

Untuk menjalankan dan mengembangkan Xinu, digunakan aplikasi virtualisasi yaitu Oracle VM VirtualBox. Virtualisasi memungkinkan kita menjalankan sistem operasi lain di dalam sistem operasi utama tanpa mengganggu konfigurasi asli komputer.

Dalam modul ini digunakan dua virtual machine (VM), yaitu:

### 1. Development-System
Development-system merupakan mesin virtual berbasis Linux Debian yang berisi:
- Source code Xinu
- Compiler untuk mengompilasi Xinu
- DHCP server
- TFTP server

VM ini digunakan untuk proses pengembangan (development), seperti mengedit source code, melakukan kompilasi, serta mengirimkan hasil build ke backend.

### 2. Backend
Backend merupakan mesin virtual yang digunakan untuk menjalankan hasil kompilasi sistem operasi Xinu. Backend akan melakukan booting melalui jaringan menggunakan protokol TFTP dan DHCP yang telah dikonfigurasi pada development-system.

Dengan adanya pembagian ini, proses pengembangan dan eksekusi sistem menjadi lebih terstruktur dan menyerupai sistem client-server.

## Guided

Berikut langkah-langkah yang dilakukan pada Modul 01:

### 1. Import Virtual Machine
- Membuka Oracle VM VirtualBox
- Klik **File → Import Appliance**
- Memilih file `development-system.ova`
- Menunggu proses import hingga selesai
- Mengulangi langkah untuk `backend.ova`

### 2. Menjalankan Development-System
- Memilih VM development-system
- Klik **Start**
- Login ke sistem Linux
- Membuka terminal untuk memastikan sistem berjalan dengan baik

📷 Screenshot Development-System Berjalan:

![Development System](images/development-system.jpg)

---

### 3. Menjalankan Backend
- Memilih VM backend
- Klik **Start**
- Backend melakukan boot melalui jaringan (PXE boot)
- Sistem mengambil file dari development-system

📷 Screenshot Backend Berjalan:

![Backend](![alt text](image.png))

---

### 4. Hasil Pengujian
Dari hasil percobaan, kedua VM berhasil dijalankan dengan baik. Development-system berfungsi sebagai server dan tempat pengembangan, sedangkan backend berfungsi sebagai mesin eksekusi Xinu.
